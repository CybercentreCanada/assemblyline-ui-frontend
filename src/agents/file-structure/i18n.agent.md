# i18n

> Applies to: `*.i18n.*.json` files and any component using `useTranslation`.

## Template

```json
{
  "drawer.empty": "No notifications",
  "drawer.title": "Notifications",
  "settings.save": "Save settings",
  "settings.delete.confirm": "Are you sure?"
}
```

```typescript
const MyComponent = memo(() => {
  const { t } = useTranslation(['notifications']);
  return <span>{t('drawer.title')}</span>;
});
```

## Rules

- All user-visible strings use `t('key')` — no hardcoded text
- Always pass namespace array: `useTranslation(['moduleName'])`
- Flat JSON structure — dot-notation keys, no nested objects
- Keys sorted alphabetically within JSON files
- Keys are relative to module namespace — no module prefix in key
- No fallback defaults: `t('key')` not `t('key', 'Default')`
- Every key must exist in both `en` and `fr` files
- One translation file per module per language: `<module>.i18n.en.json`

## Key Naming

- Dot notation for hierarchy: `feature.action` or `feature.sub.action`
- Descriptive but concise
- Group by feature area within the namespace

## Multiple Namespaces

```typescript
const { t } = useTranslation(['notifications', 'common']);
// First namespace is default
// Prefix others: t('common:button.save')
```

## Interpolation

When injecting dynamic values, use i18next `{{variable}}` syntax:

```json
{
  "adornment.reset.tooltip": "Reset {{type}} to: ",
  "items.count": "{{count}} items found"
}
```

```typescript
t('adornment.reset.tooltip', { type: fieldType })
t('items.count', { count: items.length })
```

- Use `{{variableName}}` in JSON translation values
- Pass variables as second argument to `t()`
- NO template literals (`${value}`) for injecting values into translated strings

## Forbidden

- NO template literals for user-visible text
- NO `${value}` inside translation strings — use `{{value}}` interpolation
- NO translations in shared/global files — per-module only
- NO unused translation keys
- NO reusing keys across modules
- NO `useTranslation()` without namespace argument
- NO interpolation unless strictly necessary
- NO nested JSON objects in translation files
