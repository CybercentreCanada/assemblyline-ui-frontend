# i18n Conventions

> Applies to: `*.i18n.*.json` files and any component using `useTranslation`.

## File Structure

Each module has its own translation files:

```text
<module>.i18n.en.json   ← English
<module>.i18n.fr.json   ← French
```

## Key Naming

Keys use dot notation within the module's namespace. The namespace is loaded via `useTranslation`:

```json
{
  "drawer.title": "Notifications",
  "drawer.empty": "No notifications",
  "announcement.save": "Save announcement",
  "announcement.delete.confirm": "Are you sure?",
  "systemMessage.header": "System Announcement",
  "systemMessage.none": "No announcement"
}
```

### Rules

- Keys are relative to the module namespace (no module prefix in the key itself)
- Use dots to indicate hierarchy within the namespace
- Keep keys descriptive but concise
- Group related keys by feature area
- All keys must be sorted alphabetically within their JSON file

## Usage in Components

Always pass the module namespace as an array to `useTranslation`:

```typescript
import { useTranslation } from 'react-i18next';

const MyComponent = memo(() => {
  const { t } = useTranslation(['notifications']);
  return <Typography>{t('drawer.title')}</Typography>;
});
```

### Multiple Namespaces

When a component needs keys from multiple modules:

```typescript
const { t } = useTranslation(['notifications', 'common']);
// Keys from first namespace are default
// Prefix other namespaces: t('common:button.save')
```

## Usage Rules

1. **All user-visible strings must use translation keys** — no hardcoded text
2. **No fallback default values** — use strict key usage: `t('key')` not `t('key', 'Default')`
3. **Always add both en and fr** — every key must exist in both language files
4. **Do NOT use interpolation unless necessary** — prefer separate keys over complex templates
5. **Do NOT nest JSON** — use flat dot-notation keys in a flat JSON structure
6. **Always pass namespace array** — `useTranslation(['moduleName'])`, not `useTranslation()`

## Adding New Keys

1. Add the key to `<module>.i18n.en.json`
2. Add the French translation to `<module>.i18n.fr.json`
3. Use it in your component with `t('key.name')`

## Interpolation

When you need to inject a dynamic value into a translated string, use i18next's built-in `{{variable}}` interpolation — never JavaScript template literals.

**In the JSON file:**

```json
{
  "adornment.reset.tooltip": "Reset {{type}} to: ",
  "items.count": "{{count}} items found"
}
```

**In the component:**

```typescript
t('adornment.reset.tooltip', { type: fieldType })
t('items.count', { count: items.length })
```

This ensures translations remain language-agnostic (word order may differ between languages) and keeps all user-visible text in the translation files.

## Do NOT

- Do NOT use `${value}` template literals to build translated strings
- Do NOT use template literals for user-visible text
- Do NOT put translations in shared/global files — keep them per-module
- Do NOT create translation keys that are unused
- Do NOT reuse keys across modules — each module owns its namespace
- Do NOT use `useTranslation()` without a namespace argument
- Do NOT prefix keys with the module name (the namespace handles that)
